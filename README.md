## Debriefing - Parte final
1. O que funcionou bem no processo?

   O pair review foi bom para achar corner cases, e a comunicação entre devs e o mantenedor foi clara. Além disso, os PRs continham ótimas descrições e boas devolutivas.

2. Qual foi a maior dificuldade?

   As maiores dificuldades foram pensar em todos os casos possíveis de uma entrada e lidar com as ambiguidades dos requisitos, principalmente definir a gramática utilizada nas funções que envolvem texto.

3. Qual feedback de code review foi mais útil?

   Em geral, a análise de casos de testes foi o mais útil (quando o deslocamento não era um inteiro para a tarefa 2, uso de entradas vazias e tipagem incorreta dos argumentos). Ainda assim, outro feedback positivo foi quanto à verificação do código (como no caso da tarefa 3 cuja gramática fora definida, inicialmente, de modo incorreto).

4. O que vocês fariam diferente da próxima vez?

   Criação de pull requests mais concisos, a respeito do feedback do mantenedor.
